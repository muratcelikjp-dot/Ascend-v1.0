# Product Vision

## 1. Core Vision

Real life = RPG.

Users are not just tracking habits; they are building their character. The app should make ordinary effort feel like visible character growth, not like another checklist.

Core loop:

Quest -> XP -> Attributes -> Boss Damage -> Skills -> Character Growth

Every completed action should connect back to the feeling that the user is becoming stronger, more capable, and more in control of their life.

## 2. Design Principles

- Keep the app simple at first.
- Do not add every idea immediately.
- Add features slowly and safely.
- Every feature should make the user feel motivated, understood, and in control.
- Avoid overwhelming the user.
- Prefer progressive unlocks.

## 3. Priority Features

- Daily State
- Active Buffs / Temporary Effects
- Boss System
- Character Sheet
- Progressive Unlock
- Minimum Quest / No Zero Day
- Boss Weakness
- Skill Tree
- Weekly Story Recap
- Identity System
- One Main Quest
- Journey Timeline
- Proof System
- Prestige

## 4. Later Features

- Community
- Online profiles
- Cloud save / accounts
- Built-in app blocker
- Full calorie system
- Large Cyberpunk-style visual skill tree
- Full proof/photo comparison system

## 5. Current Development Workflow

- ChatGPT acts as project manager / architect.
- Codex edits code in small safe steps.
- Test locally first.
- Commit and push to main.
- Verify GitHub Pages and iPhone PWA using the version badge.
